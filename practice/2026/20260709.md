```sql
SELECT
    Name,
    Population
FROM
    city
WHERE
    CountryCode IN
        (
            SELECT
                Code
            FROM
                country
            WHERE
                continent = "oceania"
        )
ORDER BY
    population DESC
LIMIT
    10
;
```

`IN` を `=` にすると `ERROR 1242 (21000): Subquery returns more than 1 row` というエラーになる。
このエラー（MySQLのエラー番号 1242 や Oracleの ORA-01427 など）は、「1つの値（単一の行）」が返ってくるはずの場所に、サブクエリが「2つ以上の値（複数の行）」を返してしまったときに発生します。
原因の多くは、= や < などの比較演算子を使っているか、SELECTの直後にサブクエリ（スカラ・サブクエリ）を書いていることです。

cityテーブルに都市が一件以上登録されている国が何カ国あるか求める。

```sql
SELECT
    COUNT(*)
FROM
    country
WHERE
    EXISTS
        (
            SELECT
                DISTINCT(CountryCode)
            FROM
                city
            WHERE
                city.CountryCode = country.Code
        )
;
```

```sh
+----------+
| COUNT(*) |
+----------+
|      232 |
+----------+
1 row in set (0.002 sec)
```

> `country` と `city` を入れ替えると全然結果が異なる理由

結論から言うと、**COUNT(\*)が数えてるのはFROM句に書いた方のテーブルの行数**だからだよ！
EXISTSは「存在するかどうか」を確認してるだけで、中身をカウントしてるわけじゃないの。
だから country と city を入れ替えると「何を数えてるか」自体が全く別物になっちゃうんだよね〜

## 元のクエリ vs 逆にしたクエリ

|                | FROM（数える対象） | EXISTS（フィルター用） | 意味                               |
| -------------- | ------------------ | ---------------------- | ---------------------------------- |
| 元のクエリ     | country            | city                   | 「都市データが1件以上ある国」の数  |
| 逆にしたクエリ | city               | country                | 「対応する国データがある都市」の数 |

world DBだと、country テーブルはだいたい239行、city テーブルは4000行超（4079くらい）って感じで、**そもそも母数の桁が違う**んだよね（うろ覚えだから正確な数字は自分でも確認してみてね）。

## なんでEXISTSだと「掛け算」にならないか

```
FROM country (母数はここ！)
  |
  | country を1行ずつ見ていく
  ↓
country.code = 'JPN' → EXISTS(city.CountryCode='JPN' な行があるか？) → あればYES
country.code = 'USA' → EXISTS(city.CountryCode='USA' な行があるか？) → あればYES
country.code = 'XXX' → 該当cityがなければ → NOで除外
  ↓
COUNT(*) は「YESだったcountryの行数」を数える
→ 最大でも country の総行数(239)を超えない
```

これが逆になると、母数が city（4079行）に変わるから、最大値も桁違いになる、ってわけ！

```sql
SELECT
    Name,
    CountryCode,
    Population
FROM
    city AS t1
WHERE
    Population =
        (
            SELECT
                MAX(Population)
            FROM
                city AS t2
            WHERE
                t1.CountryCode = t2.CountryCode
        )
ORDER BY
    Population DESC
LIMIT
    10
;
```
