# SQL Syntaxes 5

## グループ化

```sql
SELECT
    id,
    AVG(score) AS 平均点
FROM
    exam
GROUP BY
    id;
```

```sh
+----+-----------+
| id | 平均点     |
+----+-----------+
|  1 |   84.5000 |
|  2 |   65.5000 |
|  3 |   74.0000 |
|  4 |   74.5000 |
+----+-----------+
4 rows in set (0.003 sec)
```

---

```sql
SELECT
    class,
    MIN(birthday)
FROM
    student
GROUP BY
    class;
```

```sh
+-------+---------------+
| class | MIN(birthday) |
+-------+---------------+
| CG    | 1977-01-28    |
| Web   | 1972-03-10    |
| NULL  | 1980-05-04    |
+-------+---------------+
3 rows in set (0.000 sec)
```

注意：下のSQLではエラーになる。

GROUP BYでデータをグループ化した場合、SELECTに書けるのは以下の2パターンのみです。

- グループ化の基準にした列（例：class）
- 集計関数を適用した列（例：MIN(birthday)、COUNT(\*)、AVG(score)）

```sql
SELECT
    birthday
FROM
    student
GROUP BY
    class;
```

```sh
ERROR 1055 (42000): Expression #1 of SELECT list is not in GROUP BY clause and contains nonaggregated column 'school.student.birthday' which is not functionally dependent on columns in GROUP BY clause; this is incompatible with sql_mode=only_full_group_by
```

## グループ化した結果に条件指定

```sql
SELECT
    id,
    AVG(score) AS 平均点
FROM
    exam
GROUP BY
    id
HAVING
    平均点 >= 70;
```

```sh
+----+-----------+
| id | 平均点     |
+----+-----------+
|  1 |   84.5000 |
|  3 |   74.0000 |
|  4 |   74.5000 |
+----+-----------+
3 rows in set (0.001 sec)
```

実行順序をしっかりと理解すること。
[./1.md#実行順序](./1.md#実行順序)

```sql
SELECT
    id,
    AVG(score) AS 平均点
FROM
    exam
WHERE
    id != 3
GROUP BY
    id
HAVING
    平均点 >= 70;
```
