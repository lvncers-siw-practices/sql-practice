## 条件分岐

```sql
SELECT
    id,
    subject,
    score,
    score + 5,
    CASE
        WHEN
            score + 5 >= 90
            THEN
                "優秀"
        WHEN
            score + 5 >= 60
            THEN
                "合格"
        ELSE
            "不合格"
    END AS grade
FROM
    exam
;
```

+----+--------------+-------+-----------+-----------+

## 統合

結合するには以下２つの条件を満たす必要がある。
エラーでは `ERROR 1222 (21000): The used SELECT statements have a different number of columns` となる。

- 「列の数と順序が同じであること」
- 「対応する列のデータ型に互換性があること」

また、重複したデータが含まれると、自動的に削除される。
重複をそのまま残したい場合には、`UNION ALL` を使う。

```sql
(
    SELECT
        "S" AS 成績,
        COUNT(*) AS 人数
    FROM
        exam
    WHERE
        score >= 90
)
UNION
(
    SELECT
        COUNT(*)
    FROM
        exam
    WHERE
        score BETWEEN 80 AND 89
)
UNION
(
    SELECT
        "B",
        COUNT(*)
    FROM
        exam
    WHERE
        score BETWEEN 70 AND 79
)
UNION
(
    SELECT
        "C",
        COUNT(*)
    FROM
        exam
    WHERE
        score BETWEEN 60 AND 69
)
UNION
(
    SELECT
        "D",
        COUNT(*)
    FROM
        exam
    WHERE
        score <= 59
)
;
```

```sh
+--------+--------+
| 成績   | 人数   |
+--------+--------+
| S      |      1 |
| A      |      2 |
| B      |      2 |
| C      |      1 |
| D      |      1 |
+--------+--------+
5 rows in set (0.035 sec)
```

### 発想の違い

```txt
┌─────────┬─────────────────────────────────────────────────────────────────────┐
│ 方式     │                        やってること                                   │
├─────────┼─────────────────────────────────────────────────────────────────────┤
│ UNION   │ 「Sの行」「Aの行」…を別々に作って縦に貼り合わせる                           │
├─────────┼─────────────────────────────────────────────────────────────────────┤
│ CASE    │ examを1回舐めて、行ごとにラベル付けしてから集計                            │
├─────────┼─────────────────────────────────────────────────────────────────────┤
│ JOIN    │ 境界値テーブル(g)を先に用意して、examの各行がどの区間に落ちるかを結合条件で判定 │
└─────────┴─────────────────────────────────────────────────────────────────────┘
```

```sql
SELECT
    CASE
        WHEN score >= 90 THEN "S"
        WHEN score BETWEEN 80 AND 89 THEN "A"
        WHEN score BETWEEN 70 AND 79 THEN "B"
        WHEN score BETWEEN 60 AND 69 THEN "C"
        ELSE "D"
    END AS 成績,
    COUNT(*) AS 人数
FROM
    exam
GROUP BY
    成績
ORDER BY
    成績
;
```

```sql
SELECT
    g.成績,
    COUNT(e.score) AS 人数
FROM
    (VALUES
        ('S', 90, 100),
        ('A', 80, 89),
        ('B', 70, 79),
        ('C', 60, 69),
        ('D', 0, 59)
    ) AS g(成績, min_score, max_score)
LEFT JOIN
    exam AS e
    ON e.score BETWEEN g.min_score AND g.max_score
GROUP BY
    g.成績
ORDER BY
    g.成績
;
```

## ビュー

VIEWを複数使うとパフォーマンスが落ちるため、避けること推奨される。

```sql
CREATE VIEW
    school_view
AS
    (
        SELECT
            student.id,
            name,
            subject,
            score
        FROM
            student
        INNER JOIN
            exam
        ON
            student.id = exam.id
        ORDER BY
            student.id
    )
;
```

```sql
SELECT * FROM school_view;
```
